# Outputs Directory

## Purpose

This directory stores all generated deliverables from the M&A advisory process. Each deal gets its own subdirectory organized by output type.

## Structure

```
outputs/
└── {deal-name}/                    # e.g., "Project-Munich"
    ├── financial/                  # Financial models and analyses
    │   ├── valuation/
    │   ├── financial-models/
    │   ├── qoe-analysis/
    │   └── working-capital/
    ├── documents/                  # Transaction documents
    │   ├── cim/
    │   ├── teasers/
    │   ├── presentations/
    │   └── letters/
    ├── market-research/            # Market intelligence
    │   ├── buyer-lists/
    │   ├── comparables/
    │   └── industry-analysis/
    ├── due-diligence/              # DD materials
    │   ├── data-room-index/
    │   ├── qa-logs/
    │   └── issue-tracking/
    ├── deal-execution/             # Deal execution materials
    │   ├── loi-comparisons/
    │   ├── buyer-tracking/
    │   └── meeting-materials/
    └── archive/                    # Old versions
```

## Naming Conventions

### File Naming Standard
```
{deal-name}_{document-type}_{description}_v{version}.{ext}

Examples:
- Project-Munich_Valuation_Model_v1.0.xlsx
- Project-Munich_CIM_v2.3.docx
- Project-Munich_Strategic_Buyers_v1.5.xlsx
- Project-Munich_LOI_Comparison_v1.0.xlsx
```

### Version Numbering
- **v1.0** - Initial version
- **v1.1, v1.2** - Minor updates (typos, small changes)
- **v2.0** - Major revision (significant changes)

### Date Format (if included)
- **YYYY-MM-DD** - ISO format for dates
- Example: `Project-Munich_Teaser_2024-11-04_v1.0.pptx`

## Version Control

### Creating New Versions
- Always increment version when making changes
- Save old version before creating new one
- Document what changed in filename or notes

### Archiving Old Versions
- Move old versions to `archive/` subfolder
- Keep last 2-3 versions in main folder
- Archive monthly or after major milestones

### Tracking Changes
- Maintain changelog in knowledge base
- Note material changes in version history
- Link related document versions

## File Organization Best Practices

### 1. Use Consistent Naming
- Follow naming convention exactly
- Use underscores (not spaces or hyphens within names)
- Include version numbers always

### 2. Organize by Type
- Keep related documents together
- Use subfolders for organization
- Maintain clear hierarchy

### 3. Date Management
- Last modified date (automatic)
- Creation date (in filename if needed)
- Version date (track in knowledge base)

### 4. Access Control
- Confidential materials in secure location
- Distribution versions clearly marked
- Internal vs. external versions separated

## Integration with Agents

### Agent Output Responsibilities

**Financial Analyst:**
- Creates all Excel-based financial outputs
- Updates valuation files
- Maintains version history

**Document Generator:**
- Creates all Word/PowerPoint documents
- Converts to PDF for distribution
- Manages document versions

**Market Intelligence:**
- Creates buyer lists and research
- Updates comparable databases
- Maintains market intelligence files

**DD Manager:**
- Creates DD tracking tools
- Maintains issue logs
- Organizes data room materials

**Buyer Relationship Manager:**
- Creates buyer tracking tools
- Manages meeting materials
- Maintains LOI comparisons

**Legal Tax Advisor:**
- Creates legal/tax memos
- Reviews documentation
- Maintains legal files

### Auto-Generated Outputs

Many outputs are automatically generated and saved:
1. Agent completes task
2. Generates output file with standard naming
3. Saves to appropriate subfolder
4. Updates knowledge base with file location
5. Increments version if updating existing file

## Distribution Guidelines

### Internal Use Only
- Working drafts (.docx, .xlsx)
- Analysis files
- Internal memos
- Version history

### External Distribution
- PDFs for documents
- Final versions only
- Watermarked if needed
- Track distribution (who received what)

### Confidentiality Levels
- **Level 1 (Public):** Teaser, basic info
- **Level 2 (NDA):** CIM, detailed info
- **Level 3 (Advanced):** Data room, full details
- **Level 4 (Internal):** Working files, analyses

## Cleanup & Archiving

### Regular Cleanup (Monthly)
- Archive old versions
- Remove superseded files
- Consolidate related documents
- Update index

### Deal Closure
- Final archive of all materials
- Create deal summary document
- Export knowledge base
- Secure storage of confidential materials
